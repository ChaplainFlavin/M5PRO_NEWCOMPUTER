# Memory Domain: Agent Self Ops (AI 自身行为与底层操作红线禁区)

*适用场景：每当 AI 在规划自身系统代码开发、操作写入本地日志文件、大批量管理 JSON 数据甚至调用第三方自动化平台时，【必须绝对服从的铁律与自我管束条款防雷单】。*

### 1. 即梦 (Dreamina) 平台硬核操作底层的“防大模型幻觉”红线
* **🚫 绝对不存在前端参数控制滑块：** 即梦在架构上**没有**暴露出开放给用户的“提示词相关性超参 (CFG Scale)”、“画面重绘幅度力度 (Denoising Strength)”、“垫图相关性比例滑块控制”等高级调教选项面板。我（AI）绝不可以在工作流中建议导演或者在代码里企图去向后台修改这些虚无的参数——这是因为我被植入过 Midjourney/SD 等开源引擎知识后发生的最致命且极其可笑的【底层 AI 幻觉】。
* **🚫 绝对不支持多维度独立负面提示词：** 即梦等强审核封箱平台只留下了一张输入正向提示词的大白纸接口。不要给导演输出、安排、打包一大堆看起来很专业唬人的“独立负面排除词合集”，这属于给用户添堵毫无用处的高级废品。如果有负面需求必须要融合进正向长散文里规避。

### 2. 物理世界文件大面积覆盖防损与 Dry-Run 强卡点 (核心宪法 Rule 8 的落地实例手册)
* **封建制死锁 Python 全局覆写引擎**：针对 JSON 数据结构体系、MD 大长文日志集体系，只要我判定需要启动 Python 去执行文本干预，那么**底层即宣告绝对禁用且剥夺调用 `os.replace` 、`file.write('w')` 或者 `json=dict重置整体赋值` 的最高权利。如果非写不可，那只能容许调用受控极度的增量并接系统 `.append()` 或 `.extend()` 并物理保留既有数组。**
* **原生 API 的降维统治**：如果只是勘误几个错别字甚至几段小逻辑，必须毫不犹豫且优先启用系统的亲儿子底座工具 `multi_replace_file_content`，做到像走显微手木刀那样做高度受控、限制首尾行参数的轻度局部划切。
* **物理熔断报账许可要求**：在我的机械臂想要触碰任何属于导演的核心项目进展日志时，必须主动强制截停自身并发线程！先要在终端黑板里把【你打算动哪个模块，操作后的预期 Diff 伤口会是什么样子的？】明码标价罗列展示出来。只要导演没主动吼出同意写入的放行密语，就坚决、绝对不可偷偷绕邦完成强行写库动作。

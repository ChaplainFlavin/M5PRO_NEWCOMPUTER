# Memory Domain: Skill CV & Space Processing (视觉空间与图像控图技能)

*适用场景：当系统（AI）面临导演指派的“解决画面裁掉人物”、“背景跟原图死锁换不掉”、“人物物理比例不对”等重度空间/画幅修复需求时调用此查雷表。*

### 1. 角色三视图生成的终极防“裁腿”机制
任何 AI 绘图系统在吃参考图时，都具有极强的“画幅与透视本能继承性”。
* **裁腿的血泪病根：** 如果供我们使用的初始设计图是半身像，甚至不小心切掉了主角的脚丫，AI 的底层网络必定死死锁住这个短板画幅，任凭你文字里强调“全身图”一百遍也会被裁腿。
* **绝对空间降维解法（垫图画幅重铸法）：** 
  1. **物理造骨 架**：不用原本的裁切比例，人工或者让系统制造一张包含完整从头到脚拉满的**极致超宽屏透明/白底图片**（比如采用极端的 `21:9` 尺寸）作为最底层的骨架骨骼级垫图。
  2. **文本锁空间**：在该角色的核心提示词中，**极其强硬地植入脚部附属物件词汇**（如 `双脚完全踩在地面，穿着重型末日战靴，完全露出整只脚部细节`），用下半身实体配饰的生成算力，反向逼迫大模型在画布的底端开口重构环境与物理透视空间。

### 2. 即梦“原画换不动大背景”的越级解法：扩图边界打破法 (Outpainting)
在即梦流程中，如果对主体人物的生成非常满意（比如正好是一张 3:4 竖屏的高情绪张力图片），但背景非常狭窄或者由于吃垫图导致元素全部被局限锁死，**绝对不要去反复死磕内部套框的局部重绘，必须调用更高维度的【画幅扩图 / Outpainting】！**
1. **局部涂抹重绘的物理局限锁：** 局部涂抹重画虽然能把树变成铁柱，但因为被原本逼仄的画布空间（边框）套死了物理纵深，它**永远无法**凭空扯出具备宏大环境透视的史诗级镜头（例如深渊陷落感、万丈深海层）。画出来的场景必定小气且构图失真畸变。
2. **扩图 (Outpainting) 引擎的宏大战术欺骗：**
   * 直接把神情满意的照片喂进即梦等工具，启动**【扩图】**延展功能（譬如指令画幅直接从窄图野蛮扩展到 16:9 极宽银幕）。
   * 在配套的补充提示词填写框中，**疯狂采用【纯粹的超级环境史诗轰炸词汇】**（比如剥离所有人物描写，仅仅输入：超级开阔的无边盐湖，巨大极其深邃的深海落差柱石，被遗弃的庞然大物远景）。
   * **运行机制防坑认知：** 利用 AI 害怕画面空白、“发疯般强迫自我填充空白边缘”的数学本能，搭配广角镜头指令，大模型自然而然会把中央的人像视为定海神针抛弃重算，转而把所有的疯狂算力分配在强边距造景上生造出开阔大场面。这就是**实现“人物脸图绝对安全保底保本 + 完美无缝熔接宏大环境”的最顶流水线解法**。

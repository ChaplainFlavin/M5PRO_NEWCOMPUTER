# 家具厂智能视觉品控系统（防漏打孔）- 初期方案评估 v1.1

**讨论时间**：2026-03-08  
**文档更新时间**：2026-03-09  
**项目背景**：越南家具厂存在漏打孔问题，导致高额客诉罚款（约 200 美元/件）。目标是在低成本前提下，实现稳定、可验收、可扩展的视觉防漏孔系统。

---

## 1. 目标与范围（先把边界说清楚）

### 1.1 一期目标（必须达成）
- 缺陷范围仅限：**漏打孔**（不包含划伤、崩边、露钉等外观缺陷）。
- 单条产线落地一套可运行系统：采集、识别、报警/剔除、日志追溯。
- 具备可量化验收指标（见第 5 节）。

### 1.2 二期扩展（可选）
- 在一期稳定后，扩展到划伤、崩边、露钉等复合缺陷。
- 不新增核心硬件，仅新增数据与模型策略。

---

## 2. 技术路线（从“能跑”到“能上线”）

### 2.1 路线选择
- **PoC 阶段**：云端多模态模型 API（快速验证可行性）。
- **试产/量产阶段**：本地端侧模型（低延迟、脱网可用）。

### 2.2 对比（修正版）

| 维度 | 传统 AOI/定制 CV | 轻量 AI 方案（本项目） |
|---|---|---|
| 初始投入 | 高（通常 10 万以上） | 低（单线硬件可控制在万元内） |
| 上线速度 | 慢（需定制与大量调试） | 快（先 API 验证，再端侧迁移） |
| 数据依赖 | 强依赖大量标注 | 低样本可启动，但**非零样本** |
| 维护复杂度 | 高 | 中（需版本管理与回归测试） |
| 对网络依赖 | 低 | PoC 高，量产低（端侧部署） |

> 说明：将“零样本、零代码训练、绝不错漏”改为工程表述。实际工业场景仍需要少量数据、标定和阈值调优。

---

## 3. 系统架构（建议基线）

### 3.1 采集端（相机与光源）
- 固定俯拍工业相机（1080p/4k 视产线速度选择）。
- 配置条形 LED 无影灯，降低反光和阴影波动。
- 必做一次安装标定：视野、焦距、曝光、白平衡锁定。

### 3.2 触发端（事件生成）
- 人工站：USB 脚踏开关可用于 PoC。
- 自动线：优先使用光电传感器触发，必要时增加编码器做位置校正。
- 不建议长期依赖“键盘事件模拟”作为唯一触发链路，需保留二次确认信号。

### 3.3 推理与决策端
- PoC：云端 API 快速验证模型能力。
- 量产：端侧模型推理，输出 `PASS/NG + 置信度 + 规则命中信息`。
- 每次检测必须落盘：时间戳、班次、机台、图像 ID、结果、执行动作。

### 3.4 执行端（报警/剔除）
- 低速线：声光报警 + 人工复核取出。
- 高速线：继电器/气动剔除机构。
- 必须增加故障降级：执行机构异常时，自动切换“报警+人工复核”。

---

## 4. 关键工程难点与处理策略

### 4.1 延迟与节拍匹配
- 云端 2-5 秒延迟只适合 PoC，不适合高速产线闭环控制。
- 量产必须端侧推理，并做端到端延迟实测（采集->推理->执行）。

### 4.2 多型号切换
- 每个型号维护孔位模板（或首件基准图）。
- 切换工单时扫码加载模板，并强制执行首件确认。

### 4.3 误报与漏检平衡
- 漏检（FN）优先级高于误报（FP）。
- 根据罚款成本与停线成本设置阈值，不同产线可配置不同策略。

### 4.4 工业安全
- 任何停线/推杆动作必须接入安全联锁和急停回路。
- 禁止将 AI 结果直接作为唯一安全控制信号。

---

## 5. 验收标准（替代“20 张图 + 95%”）

### 5.1 数据集要求
- **PoC 最小集**：200 张（正常/漏孔各 100）。
- **试产验收集**：1200 张以上，覆盖白班/夜班、3 种以上光照、3 个以上型号。
- **量产监控集**：持续采样，按周回归。

### 5.2 KPI（一期建议门槛）
- 漏检率（FN）`<= 0.3%`
- 误报率（FP）`<= 2.0%`
- 端到端决策延迟 `<= 350ms`（端侧）
- 系统可用性 `>= 99.0%`（按班次统计）
- 数据追溯完整率 `= 100%`

> KPI 应在现场联合签字确认，并写入验收单。

## 6. 成本评估（CAPEX + OPEX）

### 6.1 CAPEX（单线）

| 组件 | 规格建议 | 预估成本（RMB） |
|---|---|---|
| 推理主机 | RTX 3060/4060 或同级 | 3500 - 5000 |
| 工业相机 | 1080p/4k 免驱工业相机 | 300 - 1200 |
| 光源 | 条形 LED 无影灯 | 150 - 500 |
| 触发与控制 | 光电 + 继电器 + 接线辅材 | 300 - 1000 |
| 剔除机构（可选） | 气动推杆 + 小空压机 | 1000 - 3000 |
| **小计** |  | **5250 - 10700** |

### 6.2 OPEX（月度建议预留）
- 远程巡检与日志复盘
- 模型版本回归测试
- 现场维护与备件更换
- 人员培训与 SOP 执行

> “软件费用 0 元”仅代表无授权费，不代表无工程与运维成本。

---

## 7. 实施计划（里程碑）

1. **M1 - PoC（1-2 周）**  
采样、云端验证、输出首版指标与失败样例。
2. **M2 - 端侧 Demo（2-3 周）**  
端侧推理、触发联动、最小闭环打通。
3. **M3 - 试产（2 周）**  
连续运行、统计 KPI、修正阈值与 SOP。
4. **M4 - 量产（持续）**  
版本冻结、变更流程、周报机制。

---

## 8. 风险清单与回滚策略

### 8.1 主要风险
- 光照突变导致误报波动
- 传感器抖动导致误触发
- 型号切换遗漏模板导致批量误判
- 执行机构卡滞导致漏剔除

### 8.2 回滚策略
- 任一关键 KPI 连续 2 班不达标，自动降级为“报警 + 人工复核”模式。
- 模型升级采用双版本灰度，支持一键回滚到上一个稳定版本。

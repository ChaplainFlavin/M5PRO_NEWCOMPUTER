## OBS 导播台 × 调音台防回音 × 微信直播助手推流

> **核心策略（最终定稿版）：**
>
> - OBS → 负责**画面排版** + **多场景导播切换** + **本地 MKV 录像**

## 🔌 第一步：硬件接线（全部完成后再开电脑）

### 1-A 视频线路布局

```
【导播台桌面俯视图】

  ┌─────────────────────────────────────────────────────┐
  │  (桌面左侧)                         (桌面右侧)       │
  │  ┌──────────┐  1.5m   ┌──────────┐               │
  │  │ 电脑 B   │─HDMI───►│ 分配器   │─OUT1─►(长线→投影)│
  │  │ (客户机) │         │ 1分2出   │               │
  │  └──────────┘         └──────┬───┘               │
  │                              │OUT2 1.5m           │
  │                              ▼                     │
  │  ┌──────────┐         ┌──────────┐               │
  │  │ 电脑 A   │◄USB─────│ 采集卡②  │◄HDMI          │
  │  │ (推流机) │         └──────────┘               │
  │  │          │◄USB─────┌──────────┐◄HDMI─(3-5m)──►│
  │  └──────────┘         │ 采集卡①  │      (相机)    │
  │     │                 └──────────┘               │
  │  USB声卡(红孔)◄── Zoom H6 Line OUT                │
  └─────────────────────────────────────────────────────┘

  【相机长线走向】：相机机位 ──3-5m HDMI──(沿地贴胶布)──► 采集卡①
  【大屏长线走向】：分配器OUT1 ──(现场原生/光纤线)──────► 投影/大屏
```

1. **桌面视频互连（短线）**
   - 电脑B HDMI输出 → `1.5m HDMI线` → 分配器 `IN`
   - 分配器 `OUT 2` → `1.5m HDMI线` → 采集卡② → 插电脑A USB

2. **大屏推送（长线/场地原生线）**
   - 分配器 `OUT 1` → `现场原生HDMI长线 / 自备光纤线` → 现场投影系统

3. **相机到导播台（中长线）**
   - 相机 HDMI输出 → `3-5m HDMI线`（沿地绑胶布固定）→ 采集卡① → 插电脑A USB

### 1-B 调音台接线（音频全局）

```
【调音台面板简略示意图（从左到右俯视）】

 ┌──────┬──────┬──────┬─────┬──────┬──────┬──────────────────────────────┐
 │ CH1  │ CH2  │ CH3  │ ... │ CH7  │ CH8  │        右侧接口区             │
 │(话筒)│(话筒)│(话筒)│     │ 电脑B│ 电脑B│  ┌──────────────────────────┐│
 │ ●    │ ●    │ ●    │     │ L    │ R    │  │ AUX OUT 1 ──(大光头线)──  ││
 │GAIN↑ │GAIN↑ │GAIN↑ │     │LINE  │LINE  │  │        → 电脑B 声卡红孔   ││
 │[AUX1 │[AUX1 │[AUX1 │     │[AUX1 │[AUX1 │  │ AUX OUT 2 (不用管)        ││
 │ 12点]│ 12点]│ 12点]│     │ 最左!│ 最左!│  │ REC OUT ───(莲花线)────   ││
 │      │      │      │     │      │      │  │        → Zoom H6 Line IN  ││
 │      │      │      │     │      │      │  │ MAIN OUT ──────────────   ││
 │ [推子│↕]    │      │     │ [推子│→ 0] │  │        → 现场大音箱       ││
 └──────┴──────┴──────┴─────┴──────┴──────┘  └──────────────────────────┘│
                                              │   MAIN MIX 推子 → 0刻度  │
                                              └──────────────────────────┘

  ★关键区分：话筒通道AUX1拧到12点(放行) / 电脑B通道AUX1拧到最左(归零掐死)
```

**进线（声音进调音台）：**

- 话筒 → 调音台通道 1、2、3（活动前别人已插好，卡农头）
- 电脑B USB声卡**绿孔** → `3-5m一分二大光头线` → 调音台 `LINE IN`（7/8通道）

**出线（声音从调音台出来）：**

- 调音台 `AUX OUT 1`（或SEND 1/MON 1） → `3-5m 单6.35转3.5mm线` → 电脑B USB声卡**红孔**
- 调音台 `REC OUT / TAPE OUT` → `3m 莲花线` → **Zoom H6 `Line IN`**（顶部）
- **Zoom H6 `Line OUT`**（侧面） → `<1m 短对录线` → 电脑A USB声卡**红孔（麦克风输入）**

> ⚠️ Zoom H6 在这里的作用是**电平缓冲 + 增益保护**：调音台的 REC OUT 电平较高，H6 作为中间层接收后再以合适电平输出给电脑A，防止爆音或底噪。**H6 本身不作为 USB 音频设备使用。**

## 🎛️ 第二步：调音台旋钮操作（五个动作搞定全场）

> ⚠️ 关键前提：**两台电脑插好 USB 声卡后，Windows 声音设置里「麦克风」和「扬声器」都要手动选中 USB Audio Device。**

### 动作 1：推主推子打开总闸

- 找右下角写着 `MAIN MIX` 或 `STEREO OUT` 的最长推子。
- **推到 `0` 刻度**（有加粗横线标识），不要再动它。

### 动作 2：推电脑B通道（让北京声音进会场）

- 找插了电脑B的通道推子（一般 7/8）→ 慢慢推到 `0` 刻度。
- 放个音频测试，会场音箱响起来 = 下行链路通了。

### 动作 3：防回音设置（★最重要★，决定北京领导会不会听到自己的回音）

```
【AUX 1 旋钮操作对比图】

  话筒通道 1/2/3         电脑B通道 7/8
  ┌───────────┐          ┌───────────┐
  │   AUX 1   │          │   AUX 1   │
  │     ╱     │          │           │
  │   ─┼─     │  ✅放行  │  ─┼─      │  ❌掐死
  │     ╲  12点│  现场人声 │ ◄ 最左边  │  北京回音
  │           │          │           │
  └───────────┘          └───────────┘
      拧到正中间                拧到底
   (12点方向/正上方)         (逆时针到头)
```

1. **话筒通道（1、2、3路）的 AUX 1 旋钮** → 拧到 **12 点钟（正中间）** ← 放行现场人声
2. **电脑B 通道（7/8路）的 AUX 1 旋钮** → 拧到**最左边（归零）** ← 彻底掐死回音

> **原理：** AUX 1 是一条专门送给北京的声音支路。话筒的声音要进这条路（北京才能听到现场），但电脑B（北京声音）绝不能进这条路，不然北京人会听到自己的声音延迟回来，这就是回音。

### 动作 3 验证：北京领导入会前，确认 AUX OUT 正在运作

> 不要等北京领导开口说"我这边有回音"才发现问题。用下面的方法**提前独立验证**，全程不需要北京领导在线。

**方法：在电脑B上看麦克风输入电平**

1. 确认线路已接好：调音台 `AUX OUT 1` → 电脑B USB声卡红孔。
2. 在**电脑B**上，右键任务栏右下角的小喇叭 🔊 → **【声音设置】**（或打开「控制面板 → 声音 → 录制」标签）。
3. 找到 `USB Audio Device`（电脑B的外置声卡），选中后可以看到一条**实时输入电平条**。
4. 让现场有人**对着话筒说话或拍手**。
5. **如果电平条跟着跳动 = AUX OUT 链路通了，防回音路由正在工作！**
6. **如果电平条纹丝不动** → 检查：AUX 1 旋钮有没有拧到12点方向？AUX OUT 的物理线有没有插牢？电脑B 的默认录音设备是不是选的 USB 声卡？

> ✅ **这步骤一旦确认声音进来了，北京领导入会后就直接能听到现场——你不用再担心 AUX 是否生效。**

### 动作 4：REC OUT 不用调

- 绝大部分调音台 REC OUT 是固定输出，不受推子控制。
- 会场有声 → Zoom H6 就有收到 → 电脑A 就能用。
- 只需在 Zoom H6 上查看 **输入电平指示灯**，保持绿灯为主、偶尔橙色、绝不红灯。

### 动作 5：音量平衡测试

- 对话筒说话，观察通道的 `PEAK / CLIP` 指示灯。
- **绿/橙=正常；红色持续亮=太大会爆音** → 把该通道最上方 `GAIN` 旋钮往左拧一点点。

### 🆘 调音台紧急抢救口诀

| 故障现象 | 5秒抢救动作 |
|---|---|
| 刺耳啸叫"滴——" | 先拉总推子到底 → 再把话筒通道推子拉下一半 → 慢慢推回 |
| 持续"滋滋/嗡嗡"电流声 | 拔掉笔记本充电线，或在有杂音的接口串上音频隔离器 |
| 电脑B放不出声/话筒没声 | ① 检查通道 `ON/MUTE` 键状态 ② 检查 `L/R` 路由分配按钮 ③ 检查有无开 `PHANTOM 48V`（针对电容话筒） |

## 💻 第三步：OBS 软件配置

### OBS 音轨结构（先看清楚，再动手）

```
【OBS 混音器面板示意图】

  ┌──────────────────────────────────────────────────────┐
  │                    混  音  器                         │
  ├───────────────────────┬──────────────────────────────┤
  │  桌面音频              │  全场声音_Zoom                │
  │  (Desktop Audio)      │  (手动添加的音频输入采集)       │
  │                       │                              │
  │  ████████░░░░░░░      │  ░░░░███████████░░           │
  │  ████████░░░░░░░      │  ░░░░███████████░░  ←跳动中  │
  │  ████████░░░░░░░      │  ░░░░███████████░░           │
  │                       │                              │
  │  🔇  [静音按钮]        │  🔊  [正常播放]              │
  │  ← 点这里变红/静音    │  ← 保持这条跳动绿色          │
  │                       │                              │
  │   ❌ 永久静音          │   ✅ 唯一保留                 │
  └───────────────────────┴──────────────────────────────┘
            齿轮⚙️ → 高级音频属性 → 填 Sync Offset 值
```

| OBS 混音器通道 | 对应物理来源 | 你的操作 |
|---|---|---|
| `桌面音频 (Desktop Audio)` | 电脑A 播放的一切声音（微信、浏览器、弹窗...） | ❌ **点喇叭图标 → 变红 → 永久静音** |
| `全场声音_Zoom`（手动添加） | USB声卡红孔 ← H6 Line OUT ← 调音台 REC OUT | ✅ **唯一保留，这是全场总声音** |
| 其他默认麦克风（如果出现） | 电脑内置麦克风等 | ❌ 静音或删除 |

**结论：混音器里只允许 `全场声音_Zoom` 这一条绿色跳动的音量条。**

### 3-A 添加唯一音频源

1. OBS **【混音器】** → 找到 `桌面音频` → 点喇叭 🔇 静音。
2. **【来源】** 面板 → `+` → **【音频输入采集】** → 命名 `全场声音_Zoom` → 确定。
3. 【设备】下拉 → 选择 **`USB 外置声卡`**（即 Zoom H6 Line OUT 接进来的那个，通常叫 `USB Audio Device`）。
4. 对着话筒说话 → 看到音量条跳动 = **链路通了！**

### 3-B 搭建三大导播场景


2. 【来源】`+` → **【视频采集设备】** → 命名 `现场相机` → 设备选**采集卡①**
3. 右键预览画面 → `变换` → **【比例适配屏幕】**

**场景 2：`纯看PPT_领导`（电脑B画面满屏）**

1. 【场景】`+` → 命名 `纯看PPT_领导`
2. 【来源】`+` → **【视频采集设备】** → 命名 `电脑B画面` → 设备选**采集卡②**

**场景 3：`横排画中画`（PPT主画面 + 右下角现场小窗）**

1. 【场景】`+` → 命名 `横排画中画`
2. 【来源】`+` → **【视频采集设备】** → **【添加现有】** → 选 `电脑B画面`，铺满全屏
3. 再点 `+` → **【视频采集设备】** → **【添加现有】** → 选 `现场相机`
4. 预览里把现场相机拖到右下角 + 缩小（不挡 PPT 关键字）

> **导播切换口诀：** 现场讲话 → 场景1；讲PPT → 场景2/3；领导发言 → 场景2

### 3-C 本地高清录像设置

> 这是交给客户的超清母带，全程必须开着。

1. OBS 右下角 **【设置】** → **【输出】** → 输出模式改**【高级】**
2. 点**【录像】**标签页：
   - 录像格式：`MKV`（断电不损坏）
   - 视频编码器：N卡选 `NVENC H.264`；核显选 `QuickSync`；AMD选 `AMF`；不要选 `x264`
   - 码率控制：`CQP`，级别 `22`（数字越小越清晰，22是最佳平衡点）
3. 确定保存

## 🎙️ 第四步：测算延迟数值（只测一次，填到助手里）

> 采集卡渲染比声音慢约 150ms～250ms。需要让声音"等等"画面，保证直播间观众声画同步。
> **本地录像的偏移不影响直播质量，这一步测出的数字是专门给微信助手用的。**

**操作流程：**

1. 让人站在相机前**拍手（打板）**，同时盯着 OBS 预览画面。
2. OBS 混音器 → `全场声音_Zoom` 右边的齿轮⚙️ → **【高级音频属性】**
3. **【同步偏移 (Sync Offset)】** 框 → 输入 `150`（毫秒），回车
4. 继续看拍手效果：
   - 手合上了但声音还没响 → 说明声音还在等，数值太大，**改小**
   - 声音先响了手还没合 → 说明声音超前，数值太小，**改大**
5. 调到 OBS 预览里**声画基本对上 → 记下这个数值（例：`180`）**

> ✅ 这个数值之后要填进微信助手，是直播间声画同步的关键！

## 🆘 地狱开局应急预案

### 预案 A：调音台 AUX 全线失灵（无法给北京防回音送声）

- AUX OUT 送不了声 → 改用无线麦克风接收端直接插电脑B声卡红孔（纯无线接收，不绕调音台）
- 利用腾讯会议自带的"降噪"功能在软件层面抑制回音

### 预案 B：Zoom H6 故障（电脑A没有声音）

- 应急路线：调音台 `REC OUT` → 音频隔离器 → 直接接电脑A USB声卡红孔（省去H6）
- OBS 里的音频设备不需要重新改，物理线路接好后声音自动进来

### 预案 C：微信助手找不到"声音偏移"设置

- 不同版本的助手界面可能不同，找不到时：用 OBS 的老方案（Sync Offset 大数值）做本地预览参考，声音偏移靠实际测试调优，开播前多做拍手测试

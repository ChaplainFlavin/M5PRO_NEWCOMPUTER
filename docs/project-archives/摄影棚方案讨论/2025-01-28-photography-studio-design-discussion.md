## 问题15：电脑终端乱码/字符编码问题 ⏸️ 暂停讨论

### 问题描述

用户电脑出现中文字符显示异常（乱码），疑似字符缺失或编码配置问题。

### 环境信息

- **平台：** win32
- **系统：** MSYS_NT-10.0-26100 3.6.5-22c95533.x86_64 (Git Bash/MSYS2)
- **日期：** 2026-02-01

### 初步排查

1. 检查环境变量 `LANG`、`LC_ALL` - 无输出
2. 检查 `locale` 设置 - 无输出
3. 检查 `~/.bashrc` 配置 - 无输出

### 待执行方案（待继续讨论时实施）

**方法1：Git Bash设置**
- 打开Git Bash → 窗口左上角图标 → Options → Text
- 设置 Locale: `zh_CN`
- 设置 Character set: `UTF-8`
- 勾选 `UTF-8 I/O conversion`

**方法2：环境变量配置**
在 `~/.bashrc` 添加：
```bash
export LANG=zh_CN.UTF-8
export LC_ALL=zh_CN.UTF-8
```

**方法3：Windows系统设置**
- 设置 → 时间和语言 → 管理语言设置
- 区域 → 管理 → 更改系统区域设置
- 勾选"Beta版：使用Unicode UTF-8提供全球语言支持"

**方法4：检查Git Bash版本**
```bash
git --version
locale
```

# Session Checkpoint 使用指南

## 快速开始

### 第一次使用

直接开始工作即可，skill 会自动初始化。

### 基本命令

```bash
# 保存当前进度
/checkpoint

# 恢复当前项目的进度
/resume

# 切换到指定项目
/resume 电商网站

# 查看当前状态
/status
```

## 使用场景

### 场景 1：工作中自动保存

```
你: 帮我写一个登录功能

我: [写代码...]

你: /checkpoint

我: ✅ 已保存到 CLAUDE-checkpoint.json
    - 进行中任务: 1 个
    - 最近日志: "开始实现登录功能，创建了 login.js"
```

### 场景 2：切换模型继续工作

**在 Gemini 中：**
```
你: 帮我修复这个 bug
Gemini: [debug...]
你: /checkpoint
Gemini: ✅ 已保存
```

**切换到 Claude：**
```
你: /resume
我: 📂 恢复中...
    上次工作: 修复登录 bug
    已尝试方案:
      - 检查参数传递 ✓
      - 添加 null check (无效)
    下一步: 检查异步时序问题
```

### 场景 3：多项目切换

```
# 在项目 A 工作
目录: D:\项目\电商网站
你: /checkpoint
我: ✅ 已保存到 电商网站-checkpoint.json

# 切换到项目 B
目录: D:\项目\个人博客
你: /resume
我: 📂 检测到【个人博客】项目
    "你上次做到文章列表页..."

# 或者直接指定项目
你: /resume 电商网站
我: 📂 切换到【电商网站】
    "你刚完成了登录功能..."
```

## 工作流程建议

### 推荐节奏

1. **开始任务时**：直接工作，skill 自动记录
2. **重要节点**：手动 `/checkpoint` 保存
3. **完成任务**：自动保存到 checkpoint
4. **切换模型/项目**：`/resume` 恢复

### 关键时刻保存

建议在这些时候保存：
- ✅ 完成一个功能
- ✅ 修复一个 bug
- ✅ 做出重要决策
- ✅ 准备切换模型/项目
- ✅ Token 快用完时

## 常见问题

### Q: checkpoint 文件会很大吗？
**A:** 不会。只保存关键信息（任务、日志摘要），通常只有几 KB。

### Q: 不同项目会混淆吗？
**A:** 不会。每个项目独立的 checkpoint 文件，基于目录自动识别。

### Q: Gemini 和 Claude 都能用吗？
**A:** 能。JSON 格式通用，手动执行相同操作即可。

### Q: 如果切错了项目怎么办？
**A:** 直接说 "切换到 [项目名]" 或用 `/resume [项目名]` 即可。

### Q: 需要每次都手动保存吗？
**A:** 不需要。重要节点会自动记录，但建议关键时刻手动保存。

## 技巧

### 1. 描述工作内容
```
你: /checkpoint 我刚完成了登录功能的测试
我: ✅ 已保存，附加备注: "完成登录功能测试"
```

### 2. 切换前确认
```
你: /resume 电商网站
我: 📂 读取 电商网站-checkpoint.json
    "你刚完成了登录功能..."

你: 不对，我要看个人博客的
我: 明白，切换到个人博客...
```

```

我: 📊 项目状态
    项目: 电商网站
    创建时间: 2天前
    最后更新: 10分钟前
    使用过的模型: gemini, claude

    任务:
      ✅ 已完成: 3 个
      🔄 进行中: 1 个
      📋 待办: 2 个

    最近日志:
      - 14:30 完成登录功能
      - 15:00 发现 token bug
      - 15:20 修复方案验证成功
```
